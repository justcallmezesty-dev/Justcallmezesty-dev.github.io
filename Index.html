Throwing hyper-realistic assets into a stylized world is a recipe for visual noise.

If we are completely ditching post-processing so styles can mix seamlessly — and we definitely aren't hand-painting hundreds of assets — we need a technical, procedural pipeline. This is the blueprint: intercept the art at two levels, process the raw textures before they hit the material, then lean on a robust UE5 Master Material to handle shading and edges.

01
The Megascans Fixer

The Compute Shader Texture Processor

Megascans have too much micro-detail. To fix this without opening Substance Painter, we process the textures procedurally — at runtime, on the GPU.

You can write a compute shader that is called at the start of the game. This shader analyzes your Megascans or Kitbash albedo textures and averages out colors within predefined strokes. Once processed, the compute shader feeds this simplified texture directly to your fragment shader — allowing instant use of any high-fidelity asset pack with little to no artist input.

Why this rocks

It works brilliantly on uniquely unwrapped meshes, remains fully compatible with Lumen, shadows, and reflections, and avoids the jitter associated with screen-space post-process filters.

Core Architecture

You need a “Texture Stylizer” system built from three parts — one procedural guide, one shader kernel, and one dispatcher to tie them together.

  • A Stroke Guide Map. A procedural texture (built in Substance Designer) that acts as a blueprint for brush strokes. It contains randomized shapes and directional data.
  • The HLSL Compute Shader (.usf). The logic that reads the Albedo, compares it to the Stroke Guide, and averages the colors.
  • The Dispatcher. A small C++ class or Blueprint script using the Render Dependency Graph (RDG) to run the shader and emit a new texture.

The Stroke Averaging Kernel

The goal is to eliminate micro-detail while keeping the overall color and silhouette of the Megascans asset. Use a simplified K-Means Clustering or Mean Shift across a local neighborhood. 

// StylizeAlbedo.usf — core kernel (simplified)
RWTexture2D<float4> OutStylized;
Texture2D<float4>   InAlbedo;
Texture2D<float4>   StrokeGuide;
SamplerState        LinearClamp;

[numthreads(8, 8, 1)]
void MainCS(uint3 DTid : SV_DispatchThreadID)
{
    uint2 px = DTid.xy;
    float3 seed = StrokeGuide[px].rgb;
    float3 sum = 0; float w = 0;

    // 7x7 neighborhood — weight by stroke identity
    for (int y = -3; y <= 3; ++y)
    for (int x = -3; x <= 3; ++x) {
        uint2  s = px + int2(x, y);
        float3 g = StrokeGuide[s].rgb;
        float  k = 1.0 - saturate(length(g - seed) * 4.0);
        sum += InAlbedo[s].rgb * k;
        w   += k;
    }
    OutStylized[px] = float4(sum / max(w, 1e-4), 1);
}

Implementation in UE5

Compute Shaders don't just “run” — they must be dispatched to the GPU. Use a FGlobalShader C++ class so you can call the “Paint” function from anywhere, and manage memory through the Render Dependency Graph (RDG) so the processing happens at the very beginning of the frame and never flickers.

Global Automation

To avoid doing this manually for every rock and building, build an Editor Utility Widget that iterates over selected assets, grabs their Diffuse / Normal textures, dispatches the compute shader into a transient Render Target, and spawns a stylized Material Instance that points at the new target.

Don't forget

Run Normal Maps through the same compute shader. Painterly colors with photorealistic bumpy normals will break the effect immediately — averaging the normals creates flat paint that catches light in chunks.

02
The Paint Behavior

The Stylized Master Material

Now that your textures are simplified, the Master Material needs to make them behave like wet paint reacting to light — impasto depth, oily reflections, and breakup strokes driven by a single utility texture.

The Single “Brush” Texture

Create one utility texture that packs three channels of painterly data into a single lookup. The more work this texture does, the fewer samples your material costs.

  • R — Diffuse breakup. A blurred brush shape with soft grunge, used to modulate base color and outline opacity.
  • G — Brush directions. A highly randomized normal map representing the direction of each stroke.
  • B — Detail mask. Used to drive the erosion of edges, rims, and ambient occlusion.

Cheap Impasto Depth

Run your base texture through a BumpOffset node using the brush height map. This gives your procedural strokes a thick, impasto depth — the visual illusion of paint piled onto canvas for almost no GPU cost.

“Oily” Reflections

To make the paint feel alive, use the dot(L, R) product of the light vector and reflection vector to lerp between two specular colors. To prevent the reflections from perfectly matching the diffuse strokes, swizzle your texture coordinates — pivot the normal map. As the player moves through the level, the colors dance and feel liquid and oily.

Rule of thumb

Diffuse strokes should never align with specular strokes. Offsetting one axis of the normal map UVs is the cheapest way to force them out of phase.

03
Grounding the Assets

World-Space Edges & Shadows

Without a post-process outline filter, the material itself has to handle occlusion and edge wear dynamically. Break the perfect CG shadows and edges so every object handles its own grounding procedurally.

A.

Clumpy SDF Occlusion

Sample DistanceToNearestSurface, then erode the mask with the brush alpha. Shadows pool in crevices like a dark wash of pigment instead of a smooth gradient.

B.

Smooth-Normal Edges

Bake averaged normals into vertex colors. Compare pixel normals against them to mask flat faces and only apply edge wear on real silhouettes.

C.

Camera-Aligned Rim

Use the Camera Vector's XY to project the brush texture. Strokes stay flat to the screen but pinned to the surface — no swimming, no tiling.

D.

DDX / DDY Curvature

Cross-product the partial derivatives of world position to find pixel-level flatness. Drive a subtle hue shift across otherwise boring slab faces.

The Smooth-Normal Edge Mask

Standard Fresnel is view-dependent and glowy. For structural, static edges, compare the true pixel normal against the baked smooth normal stored in Vertex Color:

Edge Mask · HLSL
EdgeMask = 1.0 − saturate ( N ₀ₚᵢₓₑₗ · V ₀ᵣₘₒᵣ )

The math yields 0 on flat faces (where normals match) and approaches 1 at sharp edges (where they diverge). Run the mask through a Power node to sharpen it, then use it as a lerp factor for edge highlights or a mask for chipped paint.

Camera-Aligned Rim Erosion

You want the silhouettes of your rocks to look like they were cut with a palette knife. Standard UVs look tiled; screen-space textures cause swimming when the camera pans. The answer is a hybrid — project the brush texture using the Camera Vector's XY components, multiplied by a scale parameter: 

// Pseudo-graph expression
float2 UV = CameraVector.xy * StrokeScale;
float  rim = Texture2DSample(BrushTex, UV).a;
float  edge = pow(1.0 - saturate(N . V), 2.0);
BaseColor = lerp(base, PaintChip, edge * rim);

Because the projection is based on camera orientation, the brush strokes stay flat relative to your screen, yet they read as pinned to the 3D surface. Pan the camera and the strokes feel like part of the paint layer at a consistent hand-drawn scale.

Face Curvature via DDX / DDY

For Kitbash3D models with massive flat surfaces, add accidental color variety so they don't look like grey slabs. Taking the cross product of the partial derivatives of world position gives you the flatness of a surface at the pixel level. Drive a small hue shift of 2% – 5% with it — enough to mimic a painter mixing slightly different blues and greys into a single stroke.

04
The Inverted Hull

Geometry-Based Outlines

If you still want a stylized outline without post-processing, use the inverted-hull method directly inside the asset setup — then break it up with a brush alpha so it never reads as a cartoon line.

The Core Concept

Create a second shell of geometry around your model that's slightly larger than the original, colored black, and rendered inside-out. You don't need to duplicate the mesh in DCC — use UE5's Geometry Scripting or simply add a second Material Slot. The shader pushes the vertices of the second slot outward along their normals and only renders the back of the faces.

Inverted Hull · Live Demo

The Smooth Normal Secret

If you expand a sharp building corner using standard normals, the outline will break at the seams. Instead, pull the baked smooth normals from Vertex Color and plug them into World Position Offset. Using smoothed normals ensures the shell inflates like a perfectly balanced balloon — continuous even on the sharpest 90° Kitbash corners.

Painterly Stroke Breakup

A solid black line looks like a cartoon; we want a messy brush stroke. Project your Packed Stroke Texture onto the outline shell using World-Aligned Textures (so the stroke size stays consistent regardless of the building's scale) and subtract its alpha from the outline opacity. The outline erodes into a dry-brush sketch that fades in and out along the edges.

Material settings

Domain: Surface · Blend Mode: Masked (or Translucent for soft brush edges) · Shading Model: Unlit · Two-Sided: on · multiply opacity by TwoSidedSign to mask front faces and show only the back of the shell.

Implementation Steps

  • Outline Master Material. Two-sided, unlit, masked. Add scalar parameters for Outline Thickness and Stroke Intensity.
  • Asset Setup. Open your Megascans or Kitbash mesh. Add a new Material Slot and apply the Outline Master. Enable Programmable Rasterizer if the asset is Nanite.
  • Global Control. Create a Material Parameter Collection. Link Outline Thickness to a global variable so you can make the world “sketchier” from Blueprint or Sequencer.

The geometry-based approach pins outlines to objects. They won't jitter like post-process lines, they accurately follow complex Kitbash geometry, and the stroke breakup keeps the hand-painted feel. Combined with the first three steps, you now have a complete, global, non-post-process painterly pipeline.

The final picture

Kitbash buildings no longer look like floating 3D models — they're built with thick, physical paint that responds to the environment's light and depth. And every Megascan dropped into the level inherits the style for free.
